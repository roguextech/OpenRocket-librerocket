Writing Custom Classes

All of the classed for the program are stored in the file ../src/rocket.py. For any class there are a few basic functions that it must have. Foremost among these is getData(self)->dict. This method returns a dict that should contain all of the parameters required to reconstruct the component when it is read in from a file. This method is important because it is called on all elements of the loaded flle when it is saved.

Example:

def getData(self)->dict:
return {
 "name":self.name,
 "type":type(self).__name__,
 "parent":self.parent,
 "data":{
  "mass":self.mass,
  "length":self.length,
  "diameter":self.diameter,
  "wallth":self.WallTh,
  "isMotorHolder":self.isMotorHolder
 }
}

This example comes from the tube class. In the retruned dict, there should be 4 fields: name,type,parent,anddata. name is used to store the name of the component, and is used in the treeview for rendering. type is an automatically generated field required for the component to be properly written to a file. parent stores the component's parent. This is used in a number of things.The key data can store any desired data so long as it is in key-value pairs. This is where all component data is stored. While these keys do not need to be in this order, it is recommended that they are kept in this order for consistency. The return type must be dict.
